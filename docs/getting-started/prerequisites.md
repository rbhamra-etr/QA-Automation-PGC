# Prerequisites

Use this checklist before running tests locally.

## Manual Software To Install

- Node.js 20+ (includes npm)
- Google Chrome
- Git (recommended)
- VS Code (recommended)

## Required

- Access to at least one target environment (QA or UAT)
- Valid test credentials for the systems you plan to test

## Troubleshooting

- If npm commands fail on a corporate Windows machine (execution policy or certificate issues), see [Windows npm + PowerShell + Zscaler Troubleshooting](windows-npm-powershell-zscaler.md).

## Access Prerequisites

- GitHub account with GitHub Copilot access (recommended for AI-assisted test authoring and maintenance)

## Keys Required For Setup

Use the following keys for setup:

```dotenv
JIRA_API_TOKEN=your_jira_api_token_here
XRAY_CLIENT_ID=your_xray_client_id
XRAY_CLIENT_SECRET=your_xray_client_secret
```

Notes:
- `JIRA_API_TOKEN` is self-generated by each user from Atlassian.
- `XRAY_CLIENT_ID` and `XRAY_CLIENT_SECRET` are admin-issued and must be requested from the Jira/Xray admin.
- Do not commit secrets to source control.

## JIRA_API_TOKEN Generation Steps

1. Sign in to your Jira/Atlassian account.
2. Open `https://id.atlassian.com/manage-profile/security`.
3. In the API tokens section, click Create and manage API tokens.
4. Click Create API token.
5. Enter a clear token name (for example, `QA Automation Jira Token`).
6. For expiration, select the maximum date available.
7. Click Create and copy the token value immediately.
8. Save it in your local `.env.qa` or `.env.uat` as:

```dotenv
JIRA_API_TOKEN=your_generated_token_value
```

Important:
- Token values are sensitive and should never be shared in chat, email, or commits.
- If your token expires, generate a new one and update your local `.env` file.
